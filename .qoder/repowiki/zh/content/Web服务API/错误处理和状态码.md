## 简介
本文件详细记录了API文档解析服务的错误处理机制和状态码规范。该系统提供了完整的错误处理策略，包括HTTP状态码、错误消息格式、异常情况处理以及客户端错误处理最佳实践。系统支持多种LLM提供商，每种提供商都有特定的错误处理逻辑和重试策略。

## 核心组件

### HTTP状态码规范
系统遵循RESTful API标准，使用标准HTTP状态码进行错误处理：

| 状态码 | 用途 | 错误场景 |
|--------|------|----------|
| 200 | 成功 | 同步解析成功、健康检查 |
| 201 | 任务创建成功 | 异步解析任务创建 |
| 400 | 请求错误 | 无效参数、文件类型不支持、文件过大、JSON格式错误 |
| 404 | 资源不存在 | 任务ID不存在 |
| 500 | 服务器内部错误 | 解析过程中的未预期异常 |

## 故障排除指南

### 常见错误场景诊断

#### 文件格式不支持
**症状**: HTTP 400错误，提示不支持的文件类型
**诊断步骤**:
1. 检查文件扩展名
2. 验证文件格式有效性
3. 确认文件未损坏

**解决方案**:
- 使用支持的文件格式（PDF、DOCX、XLSX、TXT、MD）
- 重新生成或转换文件格式

#### 文件过大
**症状**: HTTP 400错误，提示文件超过大小限制
**诊断步骤**:
1. 检查当前文件大小
2. 对比配置中的最大文件大小限制

**解决方案**:
- 减少文件大小或拆分文件
- 调整配置中的文件大小限制

#### JSON格式错误
**症状**: HTTP 400错误，提示output_schema不是有效JSON
**诊断步骤**:
1. 验证JSON语法
2. 检查JSON结构完整性

**解决方案**:
- 使用JSON验证工具检查格式
- 确保JSON字符串转义正确

#### LLM提供商错误
**症状**: HTTP 500错误，提供商调用失败
**诊断步骤**:
1. 检查API密钥配置
2. 验证网络连接
3. 确认提供商服务可用性

**解决方案**:
- 验证API密钥有效性
- 检查网络连接和防火墙设置
- 联系提供商技术支持

### 重试策略

#### 客户端重试最佳实践
```mermaid
flowchart TD
Request[发送请求] --> CheckResponse{检查响应}
CheckResponse --> Success{成功?}
Success --> |是| Complete[完成]
Success --> |否| CheckError{检查错误类型}
CheckError --> NetworkError{网络错误?}
CheckError --> ProviderError{提供商错误?}
NetworkError --> RetryNetwork[网络重试]
ProviderError --> RetryProvider[提供商重试]
RetryNetwork --> Wait[等待退避]
RetryProvider --> Wait
Wait --> Request
```

#### 重试配置建议
- **指数退避**: 1秒、2秒、4秒、8秒...
- **最大重试次数**: 3次
- **超时设置**: 60秒
- **并发限制**: 5个并发请求

- **标准化错误处理**: 遵循RESTful API标准

- **健壮的重试机制**: 提高系统可靠性
- **灵活的配置选项**: 支持不同环境的需求

建议在生产环境中结合实际使用情况进行适当的配置调整和监控设置。
