## 简介
API文档解析器是一个使用大语言模型智能解析API文档的工具，支持PDF、Word、Excel等多种格式，可输出结构化的JSON数据。项目提供了CLI命令行界面和Web服务两种使用方式，支持多种LLM提供商，包括OpenAI、Azure OpenAI、Anthropic Claude、Ollama本地模型以及自定义API。

## 系统要求
- Python >= 3.11
- 支持的操作系统: macOS, Linux, Windows

## 安装指南

### 方法一：手动安装
```bash
# 克隆仓库
git clone https://github.com/wowzhangji/doc_ai_parser.git
cd doc_ai_parser

# 创建虚拟环境
python -m venv .venv
source .venv/bin/activate  # Linux/Mac
# 或 .venv\Scripts\activate  # Windows

# 安装依赖
pip install -r requirements.txt
pip install -e .
```

## 基本使用

### 生成示例要求说明文件
```bash
api-doc-parser example-requirement -o requirement.json
```

### 解析API文档
```bash
api-doc-parser parse \
  api_document.pdf \
  --requirement requirement.json \
  --output result.json \
  --provider openai \
  --model gpt-4
```

### 使用自定义API（如vLLM）
```bash
api-doc-parser parse \
  api_document.pdf \
  --requirement requirement.json \
  --output result.json \
  --provider custom_openai \
  --api-base http://localhost:8000/v1 \
  --model my-model
```

### 查看支持的提供商
```bash
api-doc-parser providers
```

## CLI命令行使用

### 主要命令
1. **parse命令** - 解析API文档
2. **providers命令** - 查看支持的LLM提供商
3. **example-requirement命令** - 生成示例要求文件

### 关键配置选项
- `--provider/-p`: 指定LLM提供商（默认: openai）
- `--model/-m`: 指定模型名称
- `--api-base`: 自定义API基础URL
- `--api-key`: API密钥
- `--chunk-size`: 分块大小（默认: 3000）
- `--temperature/-t`: 模型温度参数（默认: 0.1）
- `--previous-result`: 增量更新的前一次结果
- `--verbose/-v`: 显示详细输出

## 智能分块器

### 功能特性
- **结构感知分块**：基于文档结构（标题、章节、API端点）进行智能分块
- **滑动窗口分割**：确保大块内容被合理分割
- **上下文保持**：自动添加相邻块的上下文信息
- **重叠缓冲**：避免信息截断，保证上下文连续性

### 分块策略
1. **语义分块**：优先按文档结构进行分块
2. **长度控制**：根据token数限制确保模型处理能力
3. **API端点保护**：确保API相关信息不被截断
4. **上下文增强**：为每个块添加相关上下文信息

## 配置选项详解

### 解析配置
- `provider`: LLM提供商（openai, azure, anthropic, custom_openai, custom_anthropic, ollama）
- `model`: 模型名称（可选）
- `api_base`: 自定义API基础URL（可选）
- `api_key`: API密钥（可选）
- `chunk_size`: 分块大小（默认: 3000 token）
- `chunk_overlap`: 分块重叠大小（默认: 200 token）
- `temperature`: 模型温度参数（默认: 0.1）
- `max_retries`: 最大重试次数（默认: 3）
- `use_cache`: 是否使用缓存（默认: True）

### 应用配置
- `max_file_size`: 最大文件大小（默认: 100MB）
- `upload_dir`: 上传目录（默认: ./uploads）
- `debug`: 调试模式（默认: False）

## 常见使用场景

### 场景一：基础文档解析
```bash
# 1. 生成要求说明文件
api-doc-parser example-requirement -o requirement.json

# 2. 解析API文档
api-doc-parser parse api_document.pdf \
  --requirement requirement.json \
  --output result.json \
  --provider openai \
  --model gpt-4
```

### 场景二：增量更新
```bash
# 1. 首次解析
api-doc-parser parse document_v1.pdf \
  --requirement requirement.json \
  --output result_v1.json

# 2. 增量更新（仅处理变更部分）
api-doc-parser parse document_v2.pdf \
  --requirement requirement.json \
  --output result_v2.json \
  --previous-result result_v1.json
```

### 场景三：自定义模型集成
```bash
# 使用vLLM或TGI等自定义API
api-doc-parser parse api_document.pdf \
  --requirement requirement.json \
  --output result.json \
  --provider custom_openai \
  --api-base http://localhost:8000/v1 \
  --model llama3-8b-instruct
```

## 故障排除

### 安装问题
1. **Python版本不兼容**
   - 确保使用Python 3.11+
   - 使用虚拟环境隔离依赖

2. **依赖安装失败**
   - 检查网络连接
   - 使用国内镜像源

### 配置问题
1. **环境变量未生效**
   - 检查.env文件格式
   - 确认环境变量命名正确

2. **LLM提供商认证失败**
   - 验证API密钥有效性
   - 检查网络连接和代理设置

### 运行时错误
1. **CLI命令执行失败**
   - 检查输入文件路径和权限
   - 验证要求说明文件格式

2. **Web服务启动失败**
   - 确认端口8000未被占用
   - 检查防火墙设置

### 性能问题
1. **解析速度慢**
   - 调整chunk_size参数（默认3000）
   - 减少并发数量
   - 检查网络延迟

2. **内存使用过高**
   - 启用缓存清理
   - 降低并发数量
   - 检查任务队列清理
